# ТЗ: перемещение плиток в расширенную белую область

## 1. Цель

Исправить поведение после расширения белой области планировки.

Если пользователь увеличил белое поле, он должен иметь возможность переносить помещения в новое пустое место справа или снизу.

## 2. Проблема

Белая область расширяется, но перемещение плиток ограничено старой рабочей зоной `0–100%`.

Из-за этого пользователь видит новую пустую область, но не может перетащить туда помещение.

## 3. Новое поведение

Граница перемещения плиток должна считаться от текущего размера белого поля:

- если белое поле шире базовой области, плитки можно двигать дальше вправо;
- если белое поле выше базовой области, плитки можно двигать дальше вниз;
- визуальный размер плиток при этом не меняется;
- координаты плиток могут стать больше 100%, если плитка размещена в расширенной части поля.

## 4. Совместимость

Не ломать:

- расширение белого поля;
- изменение размера окна просмотра;
- предупреждение выхода из родителя;
- порядок плиток по последнему взаимодействию;
- resize самих плиток.

## 5. Критерии приемки

1. После расширения белого поля плитку можно перетащить в новую пустую зону справа.
2. После расширения белого поля плитку можно перетащить в новую пустую зону снизу.
3. Плитка не масштабируется при переносе.
4. Плитка не выходит за фактические границы белого поля.
5. Если плитка вышла из родителя, предупреждение продолжает появляться.
