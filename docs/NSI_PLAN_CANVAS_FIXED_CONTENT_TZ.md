# ТЗ: расширение белой области без масштабирования плиток

## 1. Цель

Исправить поведение изменения размера белой области планировки.

При растягивании белого поля должны меняться только размеры самого поля, а существующие плитки помещений не должны менять визуальный размер и положение.

## 2. Проблема

Сейчас координаты и размеры плиток задаются в процентах от белого поля.

Из-за этого при увеличении белой области все плитки визуально растягиваются и смещаются, хотя пользователь хочет просто добавить свободное место справа или снизу.

## 3. Новое поведение

Белая область должна работать как лист:

- пользователь тянет уголок белого поля;
- белое поле становится шире или выше;
- существующие плитки остаются на прежних местах;
- существующие плитки сохраняют визуальный размер;
- справа или снизу появляется дополнительное пустое место.

## 4. Техническое правило

Для плиток вводится базовая рабочая система координат.

Координаты плиток продолжают храниться в процентах, но при отрисовке переводятся в пиксели относительно базового размера поля, а не текущего размера растянутой белой области.

## 5. Совместимость

Не ломать:

- перемещение плиток;
- изменение размера плиток;
- смену цвета плиток;
- вертикальные подписи;
- предупреждение о выходе из родителя;
- внешний уголок окна просмотра.

## 6. Критерии приемки

1. Белое поле можно растянуть за уголок.
2. Плитки при растяжении поля не увеличиваются.
3. Плитки при растяжении поля не меняют положение.
4. Появляется дополнительное пустое место поля.
5. Редактор плиток продолжает работать.
