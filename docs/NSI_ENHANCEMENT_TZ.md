# ТЗ: наполнение систем, планировки этажей и компактные техкарты

## 1. Цель

Доработать текущий прототип модуля НСИ, чтобы он лучше демонстрировал рабочий сценарий заполнения объекта:

- расширить демо-данные систем и оборудования;
- улучшить планировку каждого этажа;
- упростить отображение техкарт при большом количестве записей;
- сделать правую карточку компактнее, чтобы данные помещались в экран и не перегружали интерфейс.

## 2. Наполнение систем и оборудования

### 2.1. Требования

В разделе `Системы и оборудование` необходимо добавить больше реалистичных данных:

- общезданные системы;
- локальные системы арендатора / офиса;
- подсистемы;
- группы оборудования;
- конкретные единицы оборудования;
- оборудование без системы.

### 2.2. Минимальный состав систем

Добавить или расширить:

- электроснабжение здания;
- пожарную сигнализацию;
- СКУД здания;
- видеонаблюдение;
- вентиляцию здания;
- водоснабжение;
- канализацию;
- отопление;
- освещение общих зон;
- кондиционирование офиса;
- локальную СКС офиса;
- освещение офиса;
- питание серверной;
- кондиционирование серверной.

### 2.3. Связи

Для систем и оборудования должны быть видны связи:

- где система создана;
- где система действует;
- какое оборудование входит в систему;
- в каких помещениях размещено оборудование;
- какие техкарты связаны с системой, помещением или оборудованием.

## 3. Планировки этажей

### 3.1. Требования

В режиме `Планировка` необходимо улучшить отображение этажей:

- каждый демонстрационный этаж должен иметь свою схему;
- помещения должны быть расположены логично;
- схема должна быть похожа на понятный навигационный план, а не на случайные прямоугольники;
- помещения должны оставаться кликабельными;
- при клике должна открываться правая карточка выбранной сущности.

### 3.2. Этажи

Минимально проработать:

- `-1 этаж`;
- `1 этаж`;
- `4 этаж`;
- `7 этаж`.

## 4. Упрощение техкарт

### 4.1. Проблема

Техкарт может быть сотни, поэтому одновременный вывод нескольких больших блоков группировки перегружает экран.

### 4.2. Новое поведение

Во вкладке `Техкарты` должен быть один выпадающий список группировки:

- по помещениям;
- по оборудованию;
- по видам работ;
- по системам;
- по статусам.

После выбора группировки отображается только один список сгруппированных техкарт.

### 4.3. Требования к списку

- группы должны быть компактными;
- внутри группы показывать короткие строки техкарт;
- статус показывать бейджем;
- при клике по ТК открывать карточку справа;
- интерфейс должен оставаться удобным при десятках и сотнях записей.

## 5. Правая карточка

### 5.1. Проблема

Карточки справа сейчас занимают слишком много места и плохо помещаются в экран.

### 5.2. Требования

Сделать правую карточку компактнее:

- уменьшить отступы;
- уменьшить высоту строк;
- добавить внутреннюю прокрутку;
- ограничить высоту длинных списков;
- сохранять читаемость основных данных;
- не дублировать лишние подписи.

## 6. Ограничения

Не требуется:

- подключать backend;
- делать CAD-редактор планировок;
- менять основную структуру вкладок;
- полностью переписывать приложение;
- удалять текущие базовые сценарии создания сущностей.

## 7. Критерии приемки

Задача считается выполненной, если:

1. В системах появилось больше реалистичных демо-данных.
2. Есть общезданные и локальные системы.
3. Есть оборудование разных уровней: группа, дочерняя единица, отдельная единица, без системы.
4. У каждого демонстрационного этажа есть более проработанная планировка.
5. Помещения на плане кликабельны и открывают правую карточку.
6. Вкладка `Техкарты` использует выпадающий список группировки вместо одновременного вывода всех группировок.
7. Правая карточка стала компактнее и помещается в экран за счет прокрутки и меньших отступов.
