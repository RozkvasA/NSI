# ТЗ: свободное поле планировки и вертикальные подписи

## 1. Цель

Доработать планировку так, чтобы рабочее поле не воспринималось как жестко ограниченный прямоугольник, а само поле можно было двигать внутри области просмотра.

Дополнительно нужно улучшить подписи узких плиток: если плитка сильно вытянута вертикально, название должно разворачиваться вертикально.

## 2. Свободное поле планировки

В режиме `Планировка` внутри области просмотра должен быть отдельный слой `поле планировки`.

Пользователь должен иметь возможность:

- зажать пустое место на плане;
- сдвинуть поле планировки мышью;
- вернуть поле в исходное положение кнопкой `В центр`.

## 3. Ограничения по движению поля

В рамках статического прототипа поле двигается визуально внутри браузера.

Перемещение поля не должно менять координаты помещений. Координаты помещений остаются в процентах внутри самой планировки.

## 4. Вертикальный текст на узких плитках

Если плитка узкая и вертикально вытянута, название должно отображаться вертикально.

Правило:

- если ширина плитки меньше или равна 7%;
- и высота плитки больше ширины минимум в 1,7 раза;
- плитка получает вертикальную подпись.

## 5. Совместимость с редактором

Доработка не должна ломать:

- выбор помещения;
- открытие карточки справа;
- перемещение плиток;
- изменение размера плиток;
- изменение цвета плиток;
- иерархические слои контейнеров и дочерних помещений.

## 6. Критерии приемки

1. Планировка отображается как поле внутри области просмотра.
2. Поле можно двигать мышью за пустое место.
3. Есть кнопка `В центр` для сброса сдвига.
4. Координаты помещений при сдвиге поля не меняются.
5. Узкие вертикальные плитки показывают название вертикально.
6. Редактор плиток продолжает работать.
