# ТЗ: дефолтная планировка как на скриншоте и углы resize

## 1. Цель

Закрепить текущую пользовательскую схему планировки как дефолтную для системы и доработать resize-уголки плиток.

Планировка должна открываться сразу в виде вытянутого офиса:

- вход слева;
- верхний ряд помещений;
- верхний коридор;
- центральный блок;
- нижний коридор;
- нижний ряд помещений.

## 2. Resize плиток

В режиме редактирования у плиток должны быть resize-уголки:

- справа снизу;
- слева сверху.

Уголки должны быть менее заметными:

- меньше размер;
- ниже контраст;
- без тяжелой обводки;
- проявляются сильнее только при наведении или активном редактировании.

## 3. Поведение resize

Правый нижний угол меняет ширину и высоту плитки вправо и вниз.

Левый верхний угол меняет размер плитки через изменение левой и верхней границы:

- правый нижний край остается на месте;
- плитка может стать шире влево;
- плитка может стать выше вверх.

## 4. Критерии приемки

1. Стартовый экран остается планировкой.
2. Дефолтная планировка соответствует пользовательскому скриншоту.
3. В режиме редактирования у каждой плитки есть нижний правый resize-угол.
4. В режиме редактирования у каждой плитки есть верхний левый resize-угол.
5. Уголки не перетягивают внимание на себя.
6. Перемещение плиток, zoom и расширение белого поля продолжают работать.
