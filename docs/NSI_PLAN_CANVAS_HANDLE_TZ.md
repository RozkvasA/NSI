# ТЗ: изменение белого поля планировки через уголок

## 1. Цель

Добавить изменение размера именно белой области планировки, то есть внутреннего поля `plan-canvas`.

Ранее был добавлен уголок для изменения окна просмотра. Теперь нужен второй уголок внутри белого поля, чтобы пользователь мог расширять само поле, на котором расположены помещения.

## 2. Требования

В режиме `Планировка`:

- окно просмотра остается изменяемым внешним уголком снизу справа;
- белое поле планировки получает свой уголок снизу справа;
- при перетаскивании внутреннего уголка меняется размер поля планировки;
- координаты помещений остаются в процентах от поля;
- плитки визуально масштабируются вместе с полем;
- настройки через текстовые панели не возвращаются.

## 3. Ограничения

- Не менять родительские связи помещений.
- Не менять координаты плиток при изменении размера поля.
- Не добавлять обратно числовые панели настройки.
- Не ломать перемещение поля, редактирование плиток и предупреждение выхода из родителя.

## 4. Критерии приемки

1. У окна просмотра есть внешний уголок.
2. У белого поля планировки есть отдельный внутренний уголок.
3. Внутренний уголок меняет размер белого поля.
4. Плитки остаются на своих относительных местах.
5. Панели с числовыми настройками не отображаются.
