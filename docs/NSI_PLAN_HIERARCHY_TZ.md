# ТЗ: иерархические слои на планировке

## 1. Цель

Настроить отображение планировки так, чтобы иерархия дерева учитывалась на плане.

Родительские сущности должны отображаться как нижние контейнеры, а дочерние помещения и единицы должны отображаться поверх них.

## 2. Основное правило

Если сущность выше уровнем в дереве, то на планировке она должна быть визуально ниже по слою.

Пример:

```text
4 этаж
  Офис СОТА, левое крыло
    Open space 1
    Кабинет СОТА 01
    Переговорная СОТА 01
```

На планировке:

- `Офис СОТА, левое крыло` отображается как большая фоновая область;
- `Open space`, кабинеты и переговорки отображаются поверх нее;
- при клике по дочерней плитке открывается карточка дочернего помещения;
- при клике по фоновой области можно открыть карточку родительской зоны.

## 3. Требования к отображению

- родительские плитки должны быть визуально спокойнее дочерних;
- родительские плитки должны иметь подпись `контейнер`;
- дочерние плитки должны оставаться основными кликабельными элементами;
- порядок отрисовки должен рассчитываться по глубине в дереве;
- чем глубже сущность в дереве, тем выше она отображается на плане.

## 4. Требования к редактору

В режиме редактирования должно сохраняться текущее поведение:

- можно выбрать родительскую или дочернюю плитку;
- можно перемещать плитку;
- можно менять размер;
- можно менять цвет;
- сохранение и отмена работают для всех уровней.

## 5. Ограничения

Не требуется:

- автоматическая проверка геометрического вхождения дочерних плиток в родителя;
- сложные CAD-слои;
- ручное управление z-index пользователем.

## 6. Критерии приемки

1. Родительская зона отображается под дочерними помещениями.
2. Дочерние помещения видны поверх родителя.
3. На 4 этаже офис СОТА отображается как контейнер левого крыла.
4. Open space, кабинеты и переговорки СОТА отображаются поверх контейнера.
5. Редактор планировки продолжает работать для всех плиток.
